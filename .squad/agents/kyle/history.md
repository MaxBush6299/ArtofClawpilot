- **2026-04-27 issue #17 auth contract recommendation:** Max requested auth contract design for manual API-triggered generation (issue #17). Kyle recommended API key-based auth (MVP Tier 1) with Azure Functions + Key Vault integration. Key design decisions: (1) API key validation at function layer with per-caller rate limiting (5 requests/day, 2 concurrent, 15-min cooldown), (2) idempotency binding via `requestId` checked against GitHub `data/gallery.json` state before job trigger (prevents duplicate job waste), (3) `guidingDescription` treated as opaque Curator context (not raw prompt injection), (4) structured logging with caller identity and request metadata but excluding full description text (protect user creative content), (5) request validation rejects oversized/malformed payloads (max 1000 chars for description, 50KB payload cap), (6) orchestrator remains auth-agnostic (API layer owns all auth/validation logic). Rejection scenarios: missing/invalid API key (401), malformed input (400), rate limit exceeded (429), duplicate request (200 idempotent). Abuse prevention: daily/burst limits, input validation, idempotency enforcement, payload size cap. Future enhancements deferred to Tier 2: managed identity support, multi-tenant quotas, webhook callbacks, content moderation.
