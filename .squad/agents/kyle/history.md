- 2026-04-27 multi-run idempotency design: Max asked about supporting manual API-triggered runs in addition to scheduled daily runs without duplicates. Kyle designed backend contract introducing `runId` as primary idempotency key (format `{runDate}-{source}-{seq}`), keeping `runDate` as day-level granularity. Key decisions: each run gets unique `runId`, each image gets unique `imageId` with collision suffix, skip records indexed by `runId`, file naming supports multiple images per day, API route queues new runs with distinct `runId`, same `runId` rerun = no-op, git commits remain atomic one-per-publish. Three-phase migration: (1) add optional `runId` support backward-compatible, (2) implement API route with auth, (3) enforce uniqueness. Proposal written to `.squad/decisions/inbox/kyle-multi-run-idempotency.md` covering idempotency keys, data shape, file naming, CLI/API inputs, logs, retry behavior, and coexistence of scheduled + manual runs.
