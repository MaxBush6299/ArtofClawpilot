- 2026-04-24 hosted job git push failure diagnosed: the ACA Job reaches git push successfully (auth, clone, orchestrator run, commit all pass), but `git push` fails with exit code 128 permission denied. Root cause is GitHub App lacks Contents: Write permission — the App can clone (Read) but cannot push. No branch protection exists on main. The architecture explicitly assumes direct push to `main`, not PR workflow. Recommended fix: grant Contents: Write to the GitHub App installation, then rerun the manual job to verify. Branch+PR fallback would require infra/bootstrap changes and violates the single-transaction publish contract documented in `hosted-daily-run.md`.
- 2026-04-27 team triage consensus: GitHub App permissions gap identified. Kyle and Stan confirmed direct push to `main` is the correct architecture. Butters rejected rerunning against main; approved recovery path is to create disposable `hosted-smoke` branch, fix GitHub App permissions to Contents: Read & Write, redeploy job with `githubBranch=hosted-smoke`, and rerun Phase B proof on branch. After smoke proof passes with idempotent rerun validation, production cutover resumes with intended direct-push-to-main design. Documentation update needed: add GitHub App permission prerequisites to `docs/architecture/hosted-daily-run.md`.
